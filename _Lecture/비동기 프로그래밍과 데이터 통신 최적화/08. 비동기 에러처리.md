# 비동기 에러처리

## 1. 자바스크립트 에러핸들링 기본

- `try`: 실행하고 싶은 코드를 넣습니다. 여기서 에러가 나면 즉시 catch로 점프합니다.
- `catch`: 발생한 에러 객체를 인자로 받아 처리합니다.
- `finally`: 파일 닫기나 로딩 스피너 멈추기처럼 에러 여부와 상관없이 정리 작업이 필요할 때 씁니다.

```javascript
try {
  // 실행할 코드 (에러가 발생할 가능성이 있는 코드)
  const result = someFunction();
  console.log(result);
} catch (error) {
  // 에러가 발생했을 때 실행되는 코드
  console.error("에러가 발생했습니다:", error.message);
} finally {
  // 에러 발생 여부와 상관없이 무조건 마지막에 실행 (생략 가능)
  console.log("작업을 종료합니다.");
}
```

- `에러 던지기: throw`

```javascript
function checkAge(age) {
  if (age < 0) {
    // throw를 통해 에러 객체를 생성해서 던집니다.
    throw new Error("나이는 음수일 수 없습니다.");
  }
  return `나이는 ${age}살입니다.`;
}

try {
  checkAge(-5);
} catch (e) {
  console.log(e.name); // Error
  console.log(e.message); // 나이는 음수일 수 없습니다.
}
```

## 3. fetch 통신 에러핸들링

- `Promise의 catch()`

```javascript
fetch("https://api.example.com/data")
  .then((response) => response.json())
  .catch((error) => {
    console.error("데이터를 가져오는 데 실패했습니다:", error);
  });

// 4XX, 5XX 응답도 에러가 아님
fetch("https://httpstat.us/404")
  .then((res) => {
    if (!res.ok) {
      throw new Error(`http 오류. 상태코드 : ${res.status}`);
    }
    console.log(res.ok);
  })
  .catch((error) => {
    console.error("에러 발생! : ", error.message);
  });

// 에러 객체 만들어서 던지기
fetch("https://httpstat.us/404")
  .then((res) => {
    if (!res.ok) {
      const error = new Error("http 오류");
      error.status = res.status;
      throw error;
    }
    console.log(res.ok);
  })
  .catch((error) => {
    console.error(`에러! : ${error.message}, 상태코드: ${error.status}`);
  });
```

- `Async/Await에서의 처리`

```javascript
async function fetchData() {
  try {
    const response = await fetch("https://invalid-url");
    const data = await response.json();
  } catch (error) {
    console.log("비동기 에러 발생!", error);
  }
}
```

## 4. 자주 발생하는 에러 객체 종류

- `ReferenceError`: 존재하지 않는 변수를 참조했을 때
- `TypeError`: 데이터 타입이 맞지 않는 작업을 할 때 (예: null의 속성 읽기)
- `SyntaxError`: 문법이 틀렸을 때
- `RangeError`: 숫자가 허용 범위를 벗어났을 때
