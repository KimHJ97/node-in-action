# 인증과 비동기요청

## 1. Express 설정과 세션 쿠키

- `라이브러리 설치`

```bash
npm init -y
npm install express cookie-parser uuid
```

- `index.js`

```javascript
const express = require("express");
const cookieParser = require("cookie-parser");
const { v1: uuidv4 } = require("uuid");

const app = express();
app.use(express.json());
app.use(cookieParser());
app.use(express.static("public"));

const sessionStore = {};
app.post("/login", (req, res) => {
  const sessionId = uuidv4();
  sessionStore[sessionId] = {
    username: "홍길동",
    createdAt: new Date(),
  };
  res.cookie("user_id", sessionId, {
    maxAge: 1000 * 60 * 60 * 24 * 30,
    httpOnly: true,
    //secure: process.env.NODE_ENV === "production",
    //sameSite: "strict",
    //sameSite: "none",
  });

  res.json({ message: "Login successful" });
});

app.get("/me", (req, res) => {
  const sessionId = req.cookies.sessionId;
  const session = sessionStore[sessionId];

  if (session) {
    res.json({ message: `안녕하세요, ${session.username}님` });
  } else {
    res.status(401).json({ message: "세션이 없습니다" });
  }
});

app.listen(3000, () => console.log("Server is running on port 3000"));
```

- `public/index.html`

```html
<button id="loginBtn">로그인</button>
<button id="checkSessionBtn">세션 확인</button>

<script>
  const loginBtn = document.getElementById("loginBtn");
  loginBtn.addEventListener("click", async () => {
    const res = await fetch("/login", {
      method: "POST",
    });
    if (!res.ok) {
      return;
    }

    const data = await res.json();
    console.log(data);
  });

  const checkSessionBtn = document.getElementById("checkSessionBtn");
  checkSessionBtn.addEventListener("click", async () => {
    const res = await fetch("/me", {
      method: "GET",
    });
    const data = await res.json();
    console.log("세션 확인 응답:", data);
  });
</script>
```

## 2. 크로스도메인 인증정보 전송

서버에서는 cors 설정을 해주고, 쿠키에 대해서 secure 옵션을 true, sameSite 옵션을 none으로 해주어야 한다.

클라이언트는 fetch 요청시 credentials 옵션을 include로 설정한다.

### 2-1. 다른 도메인 서버 코드

- `라이브러리 설치`

```bash
npm install -g http-server
http-server # 서버 띄우기
```

- `others/other.html`

```html
<h1>다른 서버에서 요청</h1>
<button id="checkSessionBtn">세션 확인</button>

<script>
  const checkSessionBtn = document.getElementById("checkSessionBtn");
  checkSessionBtn.addEventListener("click", async () => {
    const res = await fetch("http://localhost:3000/me", {
      method: "GET",
      credentials: "include", // 옵션 추가
    });
    const data = await res.json();
    console.log("세션 확인 응답:", data);
  });
</script>
```

### 2-2. 기존 도메인 서버 코드

- `라이브러리 설치`

```bash
npm install cors
```

- `index.js`

```javascript
const cors = require("cors");

// 크로스도메인 허용 설정
app.use(
  cors({
    origin: "http://localhost:8080",
    credentials: true,
  }),
);
```

## 3. 토큰 방식 인증과 로컬 스토리지

- 백엔드는 express, 프론트엔드는 React 또는 바닐라JS
- `/get-token` 요청시 임의 token 발급하고 응답
- `/profile` 요청시 서버에서 token 정보를 확인하고 응답

### 3-1. 백엔드

```bash
npm install jsonwebtoken
```

- `index.js`

```javascript
const jwt = require("jsonwebtoken");
const SECRET_KEY = "secret-value";

// 토큰 발급
app.get("/get-token", (req, res) => {
  const token = jwt.sign({ username: "홍길동" }, SECRET_KEY, {
    expiresIn: "1h",
  });
  res.json({ token });
});

// 프로필 조회
app.get("/profile", (req, res) => {
  const token = req.headers.authorization.split(" ")[1];
  const decoded = jwt.verify(token, SECRET_KEY);
  res.json({ username: decoded.username });
});
```

### 3-2. 프론트엔드

```html
<button id="getTokenBtn">토큰</button>
<button id="getProfileBtn">토큰</button>

<script>
  const getTokenBtnEl = document.getElementById("getTokenBtn");
  getTokenBtnEl.addEventListener("click", async () => {
    const res = await fetch("/get-token", {
      method: "GET",
    });
    const data = await res.json();
    localStorage.setItem("token", data.token);
  });

  const getProfileBtnEl = document.getElementById("getProfileBtn");
  getTokenBtnEl.addEventListener("click", async () => {
    const token = localStorage.getItem("token");
    const res = await fetch("/profile", {
      method: "GET",
      headers: {
        Authorization: `Bearer ${token}`,
      },
    });
    const data = await res.json();
    console.log(data);
  });
</script>
```
