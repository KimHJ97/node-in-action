# Fetch API 이해와 활용

## 1. Fetch API

Fetch API는 브라우저에서 서버로 네트워크 요청을 보내고 응답을 받아오기 위해 사용하는 현대적이고 유연한 인터페이스이다.

과거에 사용하던 복잡한 XMLHttpRequest(XHR)를 대체하기 위해 등장했습니다. 가장 큰 특징은 Promise 기반으로 동작한다는 점이다. `.then()` 체이닝이나 `async`/`await`을 그대로 사용할 수 있다.

```javascript
fetch("https://api.example.com/data")
  .then((response) => {
    // 1. 응답 객체를 받음
    return response.json(); // 응답 본문을 JSON으로 파싱 (이것도 Promise 반환)
  })
  .then((data) => {
    // 2. 파싱된 데이터를 사용
    console.log(data);
  })
  .catch((error) => {
    // 3. 네트워크 에러 처리
    console.error("요청 실패:", error);
  });
```

### 1-1. Fetch 함수 파라미터

- method
  - get(조회), post(생성), put(전체 교체), patch(부분 수정), delete(데이터 삭제)
  - `get`/`delete`는 Body가 없이 보낸다.
- headers
  - `Content-Type`: 데이터 형식
  - `Authorization`: 토큰 정보 전달
  - `X-Custom-Header`: 사용자 내부 규약용
- body
  - GET/DELETE에서는 사용 하지 않음
  - `JSONstringify(data)` 주로 사용
  - JSON, FormData, Blob 등 사용 가능
- signal
  - 이미 보낸 요청을 취소할 수 있음 (요청 중단)
- credentials
  - same-origin: 같은 origin 일때만 쿠키 포함
  - omit: 포함 안함
  - include: 다른 도메인에서도 포함

```javascript
fetch(url, {
  method: "GET",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer 토큰",
  },
  body: JSON.stringify(data),
  credentials: "include",
  signal: controller.signal,
});

// signal - 요청 중단
const controller = new AbortController();
fetch(url, { signal: controller.signal });
controller.abort(); // 요청 중단
```

## 2. Fetch API 응답 처리

- **response 객체**
  - response.status: HTTP 상태 코드(200, 204, ..)
  - response.ok: 2XX 응답에 대해서만 true
  - response.headers: 응답 헤더
  - response.body: 응답 본문 (ReadableStream 형태)

```javascript
fetch(url).then((response) => {
  // Response 객체 사용
});
```

- **주의 사항**
  - `fetch()`는 네트워크 연결 자체가 실패한 경우(인터넷 끊김 등)에만 `catch()`로 이동한다. 서버가 4XX, 5XX 응답을 주어도 성공(Resolve)로 처리한다.
  - 본문 데이터를 읽어올 떄는 `response.json()` 혹은 `response.text()` 등을 사용한다.

```javascript
fetch(url)
  .then((response) => {
    // 4XX, 5XX 응답도 정상 처리된다.
    if (!response.ok) {
      throw new Error("서버 응답 에러!");
    }
    return response.json();
  })
  .then((data) => {
    console.log(data);
  });

async function getData() {
  try {
    const response = await fetch("https://api.example.com/data");

    if (!response.ok) throw new Error("데이터 로드 실패");

    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
```

## 3. Fetch API 실습

- 포켓몬 데이터의 이름과 능력치 출력
  - URL: https://pokeapi.co/api/v2/pokemon/pikachu
  - 이름: name 속성 파싱
  - 능력치: stat 속성 파싱
  - response.ok가 false이면 즉시 에러를 출력하고 return
  - 응답 header에 content-type을 콘솔에 출력

```javascript
// 1. then 이용
const url = "https://pokeapi.co/api/v2/pokemon/pikachu";
fetch(url)
  .then((res) => {
    if (!res.ok) return;
    console.log(`content-type ${res.headers.get("content-type")}`);
    return res.json();
  })
  .then((pikachuInfo) => {
    console.log(`name : ${pikachuInfo.name}`);
    pikachuInfo.stats.forEach((ability) => {
        console.log(ability.stat.name);
    });
  });

// 2. async 함수
async function fetchPikachu() {
  const url = "https://pokeapi.co/api/v2/pokemon/pikachu";
  const res = await fetch(url);
  if (!res.ok) {
    console.error("요청 실패");
    return;
  }

  console.log(`content-type ${res.headers.get("content-type")}`);
  return await res.json();
}

function printPikachuInfo(data) {
  const pikachuInfo = await fetchPikachu();

  console.log(`name : ${pikachuInfo.name}`);
  pikachuInfo.stats.forEach((ability) => {
    console.log(ability.stat.name);
  });
}
```

## 4. Promise APIs활용 (병렬 처리)

### 4-1. 직렬 처리

- 첫 번째 비동기 결과를 두 번째에 사용
- 두 개의 Request가 의존

```javascript
async function fetchOneThenAnother() {
  // 1. 목록 조회
  const res = await fetch("/datas");
  const dataList = await res.json();

  // 2. 첫 번쨰 상세 조회
  const id = dataList[0].id;
  const detailRes = await fetch(`/${id}/detail`);
  const detail = await detailRes.json();

  console.log(detail);
}
```

<br/>

### 4-2. 병렬 처리 - 실행 순서가 중요하지 않는 경우

```javascript
const aPromise = fetch("/a")
  .then((res) => res.json())
  .then((data) => doSomethingA(data));

const bPromise = fetch("/b")
  .then((res) => res.json())
  .then((data) => doSomethingB(data));
```

<br/>

### 4-3. 병렬 처리 - 모든 병렬 작업이 완료된 이후 처리

```javascript
Promise.all([
  fetch("/a").then((res) => res.json()),
  fetch("/b").then((res) => res.json()),
])
  .then(([dataA, dataB]) => {
    doSomethingA(dataA);
    doSomethingB(dataB);
  })
  .catch((error) => {
    console.error("요청 중 하나라도 실패:", error);
  });

// async 함수로 변환
async function fetchData() {
  const [dataA, dataB] = await Promise.all([
    fetch("/a").then((res) => res.json()),
    fetch("/b").then((res) => res.json()),
  ]);

  doSomethingA(dataA);
  doSomethingB(dataB);
}
```

### 4-4. 기타 Promise 함수들

- `Promise.all()`
  - 모두 완료된 결과를 확인.
  - 하나라도 실패하면 전체가 reject
- `Promise.allSettled()`
  - 모두 완료된 결과 확인.
  - 성공/실패 상관없이 끝날떄까지 대기함.
  - 모두 실패해도 reject 되지 않음
- `Promise.race()`
  - 제일 먼저 끝난 것을 사용
  - 가장 먼저 끝난 것이 실패하면 즉시 reject
- `Promise.any()`
  - 제일 먼저 끝난 것을 사용
  - 모두 실패해야 reject

## 5. 파일업로드와 멀티파트

Image, PDF, 영상 들은 바이너리 파일로 전송한다. 바이너리 데이터를 전송할 때 보통 `multipart/form-data` 포맷을 사용한다. 이것은 boundary라는 경계 문자열을 기준으로 구분한다.

```html
<form id="uploadForm">
  <label
    >이미지 업로드: <input type="file" name="image" accept="image/*"
  /></label>
  <br />

  <label>제목: <input type="text" name="title" /></label>
  <br />

  <button type="submit">업로드</button>
</form>

<script>
  const form = document.getElementById("uploadForm");

  form.addEventListener("submit", async (e) => {
    e.preventDefault();
    const formData = new FormData(form);

    try {
      const res = await fetch("/upload", {
        method: "POST",
        body: formData,
      });
    } catch (err) {
      alert("업로드 실패");
      console.error(err);
    }
  });
</script>
```

- **필드 직접 추가하기**
  - append 메서드를 통해서 key-value 구조로 추가한다.

```javascript
const form = document.getElementById("uploadForm");
const formData = new FormData(form);
formData.append("username", "홍길동");
formData.append("age", "20");
const file = fileInput.files[0];
formData.append("image", file);

fetch("/upload", { method: "POST", body: formData });
```

### 5-1. 서버 코드 (파일 업로드)

```javascript
const express = require("express");
const multer = require("multer");
const path = require("path");

const app = express();
const PORT = 3000;

// 업로드 디렉토리 생성
const uploadDir = path.join(__dirname, "uploads");
const fs = require("fs");
if (!fs.existsSync(uploadDir)) {
  fs.mkdirSync(uploadDir, { recursive: true });
}

// Multer 설정
const upload = multer({ dest: uploadDir });

// 이미지 업로드 API
app.post("/upload", upload.single("image"), (req, res) => {
  const fileUrl = `http://localhost:${PORT}/uploads/${req.file.filename}`;
  res.json({ url: fileUrl });
});

// 업로드된 파일 제공
app.use("/uploads", express.statis(uploadDir));
app.use(express.static("public"));

app.listen(PORT, () => {
  console.log(`서버 실행: http://localhost:${PORT}`);
});
```
